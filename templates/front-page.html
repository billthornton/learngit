Glossary

Repository
A git repository is a location where the codebase, and metadata about that codebase is stored. Repositories in git are decentralised, meaning there does not need to be one central authoritive repository. Each of your local codebase is a full repository, meaning that you can carry even if the remote repository is corrupted or fails.
git clone
This command will clone a selected repository, it's code, commits and other metadata.
HEAD
git status
Shows the current status of your local git repository. Such as what files you have modified, renamed or deleted.
git log
Shows a history of commits in your git reporitory. Both those you have made locally and those done by other developers.
git commit
A commit in git is a a set of changes to one or more files. Git will compare your local files to the version in the last commit, and work out if there has been any changes. The changes could be single lines, or many lines, or even operations such as delete or rename. You may then commit these changes so they can be applied to other developers codebases when they next pull down changes.
git push
To share your changes with other developers, you need to share your commits by pushing them. For each of use, you will most commonly be pushing to a central master repository that other developers, or servers may connect to to get the latest set of changes.
git pull
